Create `src/main/services/instagramAuthService.ts` following the gmailAuthService.ts pattern exactly:

Key implementation details:
1. Use Facebook OAuth with redirect to `https://www.facebook.com/connect/login_success.html`
2. Intercept redirect using BrowserWindow `will-redirect` event (not loopback server like Gmail)
3. Request scopes: `instagram_basic`, `instagram_manage_messages`, `pages_show_list`, `pages_manage_metadata`, `pages_read_engagement`
4. Exchange short-lived token for long-lived token (60 days)
5. Get Page Access Token and Instagram Business Account ID from `/me/accounts`
6. Store tokens using electron-store + safeStorage (matching Gmail pattern)
7. Store account info (username, name) for display

Class structure:
```typescript
class InstagramAuthServiceClass {
  private pageAccessToken: string | null = null;
  private pageId: string | null = null;
  private instagramBusinessAccountId: string | null = null;
  private accountInfo: InstagramAccountInfo | null = null;

  constructor() { this.loadStoredTokens(); }

  // Public methods
  async authenticate(): Promise<InstagramAccountInfo>
  isAuthenticated(): boolean
  getAccountInfo(): InstagramAccountInfo | null
  getPageAccessToken(): string  // throws if not authenticated
  getPageId(): string  // throws if not authenticated
  getInstagramAccountId(): string  // throws if not authenticated
  disconnect(): void

  // Private methods
  private loadStoredTokens(): void
  private saveTokens(tokens: InstagramTokens, accountInfo: InstagramAccountInfo): void
  private loadTokens(): InstagramTokens | null
  private loadAccountInfo(): InstagramAccountInfo | null
  private clearTokens(): void
  private async exchangeCodeForToken(code: string): Promise<InstagramTokens>
  private async initializeFromToken(accessToken: string): Promise<InstagramAccountInfo>
}

export const instagramAuthService = new InstagramAuthServiceClass();
```

OAuth flow specifics:
- Create BrowserWindow for auth (600x800)
- Use `will-redirect` to intercept Facebook success page
- Parse code and state from URL
- Exchange code -> short-lived token -> long-lived token
- Call `/me/accounts` to get Page with instagram_business_account
- Store page access token (not user token) for API calls
- Handle errors: no Instagram Business Account, user cancelled, etc.

Environment variables: `FACEBOOK_APP_ID`, `FACEBOOK_APP_SECRET`

Use electron-store with name `'instagram-auth'` (separate from gmail-auth).
